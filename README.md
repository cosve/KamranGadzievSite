# Сайт Кямрана Гаджиева

Небольшой сайт-портфолио для ведущего мероприятий. Внутри главная страница, отдельные страницы под свадьбу, юбилей, мероприятия, контакты и политику обработки персональных данных.

Проект сделан на Next.js App Router. Тексты вынесены в словарь, поэтому сайт работает на трех языках: русском, азербайджанском и английском. Для SEO используются отдельные URL:

- `/` — русский
- `/az` — азербайджанский
- `/en` — английский

То же правило работает для внутренних страниц, например `/wedding`, `/az/wedding`, `/en/wedding`.

## Что есть в проекте

- адаптивная главная страница с аудиоплеером, видео, галереей и формой заявки;
- страницы услуг: свадьба, юбилей, мероприятие;
- страница контактов;
- страница политики обработки персональных данных;
- отправка заявок в локальный файл и в VK-бота;
- языковой переключатель RU / AZ / EN;
- SEO-разметка: canonical, hreflang, sitemap, robots, Open Graph, Twitter Cards и JSON-LD;
- лоадер при переходах между страницами.

## Запуск

Установить зависимости:

```bash
npm install
```

Запустить dev-сервер:

```bash
npm run dev
```

Открыть сайт:

```text
http://localhost:3000
```

Проверить проект перед выкладкой:

```bash
npm run lint
npm run build
```

## Переменные окружения

Локальные настройки лежат в `.env.local`. Этот файл не нужно коммитить: там находятся контакты и токен VK-бота.

Пример:

```env
NEXT_PUBLIC_SITE_URL=https://kamrangadzhyev.com

NEXT_PUBLIC_CONTACT_EMAIL=mail@example.com
NEXT_PUBLIC_CONTACT_PHONE=+7 (000) 000-00-00
NEXT_PUBLIC_CONTACT_PHONE_HREF=+70000000000

NEXT_PUBLIC_INSTAGRAM_URL=https://example.com
NEXT_PUBLIC_VK_URL=https://vk.com/example
NEXT_PUBLIC_YOUTUBE_URL=https://youtube.com/@example
NEXT_PUBLIC_TIKTOK_URL=https://rutube.ru/channel/example

VK_BOT_TOKEN=replace_with_vk_group_token
VK_NOTIFY_USER_ID=replace_with_vk_user_id
VK_API_VERSION=5.199
```

`NEXT_PUBLIC_SITE_URL` важен для SEO: от него строятся абсолютные ссылки в sitemap, canonical и JSON-LD.

## Заявки

Форма заявки находится в общем компоненте `components/FormSection.tsx`. После отправки данные:

- записываются в `data/leads.jsonl`;
- отправляются владельцу сайта через VK API;
- возвращают пользователя на страницу с параметром успешной отправки.

В заявке используются только имя и телефон. Телефонное поле не привязано к `+7`, потому что клиенты могут быть из разных стран.

## Контент и картинки

Основные изображения лежат в `public`.

Для страниц услуг сейчас используются такие наборы:

- свадьба: `Wedding1.jpeg`, `Wedding2.jpg`, `Wedding3.jpg`;
- юбилей: `Birthday1.jpeg`, `Birthday2.jpg`, `Birthday3.jpg`;
- мероприятия: `Event1.jpg`, `Event2.avif`, `Event3.jpg`.

Если меняешь изображение с тем же именем и в браузере видна старая версия, стоит очистить кэш Next Image в `.next/dev/cache/images` или временно поменять имя файла.

## Переводы

Все основные тексты собраны в `lib/translations.ts`. Языковая логика лежит в `lib/i18n.ts`, а SEO-хелперы — в `lib/seo.ts`.

Если добавляется новая страница, нужно не забыть:

- добавить тексты для `ru`, `az`, `en`;
- добавить ссылку в `sitemap.ts`, если страница должна индексироваться;
- проверить canonical и hreflang через `npm run build`.

## VK-бот

Для отправки заявок используется метод `messages.send` VK API. Нужен токен группы и id пользователя, которому бот будет писать о новой заявке.

Токен нельзя хранить в README, отправлять в чатах или коммитить в репозиторий. Если токен уже где-то засветился, его лучше перевыпустить в настройках VK.

## Структура

```text
app/          страницы, metadata, sitemap, robots, manifest
components/   интерфейсные компоненты
lib/          переводы, SEO и i18n-хелперы
public/       изображения, аудио, видео и SVG
data/         локальные заявки, создается при отправке формы
```

## Перед выкладкой

Минимальный чек-лист:

```bash
npm run lint
npm run build
```

После выкладки стоит проверить:

- открываются ли `/`, `/az`, `/en`;
- работает ли форма заявки;
- доступен ли `/sitemap.xml`;
- доступен ли `/robots.txt`;
- корректно ли подставлен `NEXT_PUBLIC_SITE_URL` на хостинге.
